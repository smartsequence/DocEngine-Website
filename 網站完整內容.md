### Doc Engine 四步驟流程

**標題**：簡單四步驟，產出專業風險評估報告

1. **填寫評估問卷**
   - 針對系統交付情境設計的結構化問卷
   - 涵蓋技術債務、文件完整度、團隊成熟度等維度

2. **AI 智慧分析**
   - 自動產生風險指紋與成熟度量化評分
   - 識別潛在風險點與改善建議

3. **產出專業報告**
   - 風險雷達圖視覺化呈現
   - 詳細分析說明與改善建議
   - PDF 格式，可直接下載使用

4. **多元應用場景**
   - 內部檢討會議資料
   - 系統交接文件
   - 改寫評估提案
   - 對外簡報或補助案佐證

## ⚙️ Features（功能介紹）

### Hero 區塊

**標題**：Doc Engine 服務說明

**副標題**：
Doc Engine 是一套系統風險評估與報告產出服務，結合問卷量化、AI 分析與視覺化報告，協助團隊有效識別潛在風險與評估改寫成本。

---

### 適用對象

**誰需要 Doc Engine？**

**系統專案經理（PM）**
- 系統交接前需要完整的風險總覽
- 對外簡報或向主管報告時需要數據支撐
- 評估是否需要投入改寫資源

**系統分析師（SA）**
- 快速診斷遺留系統的健康狀況
- 產出結構化的系統評估報告
- 協助制定改善優先順序

**技術主管 / 架構師**
- 準備改寫的舊系統評估
- 技術債務量化與可視化
- 提供給利害關係人的風險說明

**政府機關 / 企業採購**
- 符合採購規範的正式報告
- 可開立發票與合約
- 專業的風險評估文件

**1. 問卷量化評估** ✅

### 實際應用場景

**場景 1：系統交接**
- 問題：舊系統要交給新團隊，但風險不明
- 解決：填寫問卷 → 產出風險報告 → 交接會議使用

**場景 2：改寫評估**
- 問題：老系統該不該改寫？成本如何？
- 解決：量化現況 → 評估技術債 → 提供決策依據

**場景 3：專案報告**
- 問題：需要向主管或客戶說明系統現況
- 解決：專業報告 → 視覺化呈現 → 提升信賴度

**場景 4：補助案申請**
- 問題：申請政府補助需要現況佐證
- 解決：正式報告 → 附發票證明 → 符合審查要求

### FAQ 專區（可收合）

**Q1：是否可以開立正式發票？**

是的，智序資訊工作室為正式登記之商號（統一編號：12345678），所有付費方案均可開立合法發票，符合政府採購與企業報帳需求。

**Q2：付款後何時可以收到報告？**

完成問卷填寫後，系統將自動產生報告。一般情況下，報告會在當天內產出，您可以立即下載 PDF 檔案。

**Q3：我的資料會上傳到雲端嗎？資料安全如何保障？**

您填寫的問卷資料會安全儲存於我們的伺服器，採用加密傳輸與儲存。我們不會蒐集您的客戶系統原始程式碼或敏感資料，僅針對問卷答案進行分析。所有資料處理符合個人資料保護法規範。

**Q4：可否接受企業包月或客製化合作？**

可以。如果您有特殊需求（例如：大量報告、客製化分析維度、教育訓練等），歡迎透過聯絡表單或 LINE 與我們洽詢，我們將提供專屬報價。

**Q5：試用方案與付費方案的差異？**

試用方案可以體驗完整的問卷流程，並預覽部分報告內容，但無法下載完整 PDF 檔案。付費方案則可以下載完整報告，且資料永久保存。

**Q6：如果不滿意可以退款嗎？**

依據消費者保護法，我們提供 7 天鑑賞期。如果您對報告品質不滿意，可在購買後 7 天內申請退款（需扣除已產生的報告成本）。

**Q7：月度方案的報告數量可以累積嗎？**

當月未使用完的報告數量不會累積至下個月。如果您預期某月會產生較多報告，建議選擇年度方案或聯絡我們討論彈性方案。

**Q8：可以升級或降級方案嗎？**

可以。您隨時可以升級至更高階方案。如需降級，請在當期結束前 7 天告知，將於下期生效。
